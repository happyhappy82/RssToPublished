# Claude Code 규칙

## 개발 스타일

- **병렬 처리 선호**: 독립적인 파일들은 최대한 병렬로 생성하여 효율성 극대화
- **단계별 진행**: 프로젝트 초기화 → 설정 파일 → 라이브러리 → 컴포넌트 → 페이지 → 콘텐츠 순서
- **환경변수는 마지막에 처리**: 개발 완료 후 환경변수 설정
- **TodoWrite 활용**: 진행 상황을 체계적으로 추적하고 관리
- **빌드 검증 필수**: `npm run build`로 프로덕션 빌드 테스트 완료 후 보고

## 개발 워크플로우

1. 기능 개발 요청 시: 요청 분석 → 독립된 기능들로 분해
2. 자동 판단: 독립 가능하면 Task 병렬 실행, 순서 필요하면 순차 실행
3. 각 Task 완료 시 자동 머지

### Task 분리 기준
- 파일이 겹치지 않으면 → 병렬
- 같은 파일 수정 필요하면 → 순차
- 의존성 있으면 → 순차

### 자동 머지 규칙
- 테스트 통과 시 → 자동 머지
- 실패 시 → 사람에게 보고
- 충돌 발생 시 → 중단 후 보고

## 코드 스타일 가이드

- TDD 방식 사용
- 함수는 50줄 이하로 작성
- 테스트 명령어: `npm test`
- 린트 명령어: `npm run lint`

## 프로젝트 폴더 구조

- `/src/features` - 기능별 모듈
- `/src/components` - 공통 컴포넌트

## 기능 구현 시 7개 Task 자동 분리

새 기능 요청 시 다음 7개 Task로 자동 분리:

1. **Component** - 메인 컴포넌트
2. **Styles** - CSS
3. **Tests** - 테스트
4. **Types** - 타입 정의
5. **Hooks** - 커스텀 훅
6. **Integration** - 라우팅, import
7. **Config** - 설정 파일

각 Task 독립 실행 후 완료 순서대로 자동 머지

## 서브에이전트 위임 규칙

| 상황 | 위임 대상 |
|------|-----------|
| 코드 작성 | developer 에이전트 |
| 코드 완성 후 | reviewer 에이전트 (자동) |
| 테스트 실패 | tester 에이전트 (자동) |
| 보안 이슈 | security-auditor 에이전트 (자동) |
